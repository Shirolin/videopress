# 全局规则
- 全场景（回复/计划/报告）：仅限简中（变量/库名除外）。
- 遵循第一性原理。

# Git 规范
- 格式：`<type>(<scope>): <subject>`（Angular 规范）
- 主题语言：中文。
- 类型：feat|fix|docs|style|refactor|perf|test|chore
- Scope：必填。

# 行为准则

减少 AI 编码常见错误的行为准则。可与项目级指令合并使用。

## 1. 先想清楚再写代码
- 明确陈述假设。不确定时先问清楚。
- 如果存在多种解读，列出来让用户选，不要默默做决定。
- 如果有更简单的方案，需要先指出。
- 如果有不清楚的地方，先说明困惑点。

## 2. 简洁优先
- 只写解决问题所需的最少代码，不实现未要求功能。
- 单次使用代码不过度抽象。
- 不加未被请求的灵活性或可配置性。

## 3. 外科手术式修改
- 只改与目标直接相关的文件和代码。
- 只清理本次修改引入的无用代码。
- 保持现有代码风格。

## 4. 以目标驱动执行
- 先定义可验证的验收标准。
- 多步骤任务先列步骤和验证方式。
- 每轮必须有明确的退出关卡。

## 5. Vibe Coding 护栏
- 每个可验证功能单元完成后立即提交。
- 修改前先加载相关上下文。
- 多会话项目必须维护本文件。
